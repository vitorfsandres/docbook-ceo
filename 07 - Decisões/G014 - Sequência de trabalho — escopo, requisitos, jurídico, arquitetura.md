# G014 — Sequência de trabalho: escopo → requisitos → jurídico → arquitetura

**Decisão:** Adotar sequência de desenvolvimento: (1) congelar o **escopo do MVP** a partir do app demo; (2) Igor gera 1ª versão dos **requisitos** com IA, validada por Vitor; (3) **validação jurídica** dos requisitos; (4) só então definir a **arquitetura** e o plano de entregas. Metodologia híbrida (Scrum/Kanban adaptado).

**Racional:** Requisitos claros são a base para validação jurídica, arquitetura, testes e documentação — congelar o escopo antes de codificar evita retrabalho e estouro de custo.
